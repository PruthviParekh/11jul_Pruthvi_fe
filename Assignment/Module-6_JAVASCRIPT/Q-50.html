What is the drawback of declaring methods directly in JavaScript objects?

1>  Memory Usage :

- Every object instance gets its own copy of each method. If you have many instances of an object, this can lead to higher memory usage
  compared to using prototypes where methods are shared among all instances.

2>  Performance Impact :

- Directly declared methods may impact performance, especially in scenarios where you have many instances or frequent method calls.
  Prototype methods can be more optimized by the JavaScript engine.

3> Difficulty in Managing Updates :

- If you need to update or extend the behavior of an object's methods after creation, you would have to update each instance individually.
  This can lead to code duplication and maintenance issues.

4> No Encapsulation :

- Methods declared directly in objects do not provide encapsulation or private members. All properties and methods are publicly accessible
  , which can lead to potential issues with data integrity or accidental overwriting.

5> Prototypal Inheritance Limitations :

- If you later decide to refactor your code to use inheritance (prototypal inheritance in JavaScript), methods directly declared in
  objects may not integrate well with inherited behavior, as they are not part of the prototype chain.
